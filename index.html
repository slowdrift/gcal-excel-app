1. 🎯 このアプリでできること

Googleカレンダーに登録された予定を、A4横の週間スケジュール表（Excel形式）として出力できる Web アプリです。インストール不要で、ブラウザだけで動作します。

主な特徴

  • 9:00〜17:00 の15分単位グリッドで1週間を表示
  • 月〜金は5列、土日は週末2行に集約（A4横1ページ）
  • 最大3つのカレンダーを結合して1つの日程表に集約可能（APIモード）
  • イベントの自動色分け（業務報告・移動・本会議など）
  • カレンダー説明欄から議事項目を自動抽出（業務報告・代表質問・一般質問など）
  • 場所欄の自動補完（「面会」「来客」→応接室、「幹部会」→第二応接室のみ）
  • プライバシー配慮の自動変換（「○○さん／氏／様」→「来客あり」、特定キーワード→「予定あり」）
  • 終日予定の「○○運転手」「○○運転手対応」で運転手名を当日のみ上書き
  • 祝日・年末年始閉庁日（12/29〜1/3）は運転手非表示
  • 印刷時は1週=1ページに自動最適化
  • 処理ログで除外・変換・誤植の可能性を一覧表示（詳細デバッグも切替可）

想定ユーザー

副知事・部長などの週間日程表を作成する秘書・庶務担当者を主に想定しています。

2. 📋 事前準備

このアプリには 3つのモード があります。file://で開く（HTMLをダブルクリック）方法なら ②/③ が確実、サーバー経由なら全モード利用可能です。

① APIモード（OAuth・最も自動的）

Googleアカウントでログインし、リアルタイムにカレンダー情報を取得します。

初回のみ必要な準備：

  1. Google Cloud Console でプロジェクトを作成
  2. 「APIとサービス」→「ライブラリ」で Google Calendar API を有効化
  3. 「認証情報」→「認証情報を作成」→ OAuth 2.0 クライアントID
    • アプリケーションの種類：「ウェブアプリケーション」
    • 承認済みの JavaScript 生成元：アプリを開く URL を登録
  4. 発行されたクライアントID を Step1 の入力欄に貼り付け

メリット：常に最新の予定が取得できる、最大3つのカレンダーを結合可能　注意：file://（HTMLをダブルクリック）では OAuth が動きません。http(s)://サーバー経由が必要。

② ICSファイルモード（オフライン可・どこでも動く）

Googleカレンダーから .ics ファイルをエクスポートして読み込みます。

手順：

  1. Googleカレンダーの該当カレンダー設定で「カレンダーのエクスポート」をクリック
  2. ダウンロードした zip を解凍し、.ics ファイルを取得
  3. アプリの ICSファイルモードで「ICSファイルを選択」からこのファイルを選択

メリット：API設定不要・すぐに使える・file://でも動作　デメリット：ICS取得時点までの予定のみ反映

③ ICS URLモード（自動取得・API設定不要）

GoogleカレンダーのiCal 形式の限定公開URLを直接指定して取得します。OAuth不要。

手順：

  1. Googleカレンダーの該当カレンダー設定で「カレンダーの統合」セクションを開く
  2. 「iCal 形式の限定公開 URL」をコピー（末尾が .ics）
  3. アプリの ICS URLモードに貼り付け→「🌐 URLから読み込む」
  4. 必要に応じて「💾 このブラウザに URL を保存する」をチェックしておく

メリット：API設定不要・常に最新を取得　注意：file://からはブラウザの CORS 制約で失敗することがあります。失敗時は ②「ICSファイルモード」をご利用ください。

3. 🚀 基本の使い方

Step 1：データ取得方法を選択

画面上部の3つのタブから方法を選びます：🔑 Google API 📁 .icsファイル 🌐 ICS URL

🔑 APIモードの場合：

  1. クライアントID を入力欄に貼り付け（必要に応じて「📌 保存」）
  2. 「🔑 Googleで認証する」をクリック
  3. Googleアカウントでログイン認証

📁 .icsファイルモードの場合：

  1. 「📂 ファイルを選択」で .ics ファイルを選択
  2. 必要に応じて「📂 追加ファイルを選択」で2つ目の .ics を追加
  3. 「✅ 読み込む」をクリック

🌐 ICS URLモードの場合：

  1. Googleカレンダーの「iCal 形式の限定公開 URL」を貼り付け（最大2つ）
  2. 必要に応じて「💾 このブラウザに URL を保存する」をチェック
  3. 「🌐 URLから読み込む」をクリック
  4. ※失敗時は CORS の説明＋対処法（ICSファイルモードへの誘導）が画面に表示

Step 2：期間とカレンダー選択

  1. カレンダー選択（APIモードのみ）：使いたいカレンダーを選ぶ
    • 1つ目（メイン）：プリセット保存可（📌 マークから設定）
    • 2つ目（追加・任意）：プリセット保存可
    • 3つ目（追加・任意）：プリセット保存なし（毎回手動選択）
  2. 開始日：日付ピッカーで月曜日を選択（月曜以外を選んでも自動補正されます）
    • ◀ ▶ ボタンで前後の週に移動可能
  3. 週数：1〜6週・全期間・🧪 テストモードから選択
    • 🧪 テストモード：開始日=2025-12-29／週数=全期間 を自動セット（年度をまたぐ動作確認に便利）
  4. 「📅 予定を取得」（APIモード）または「📅 期間で絞り込む」（ICS系）をクリック

Step 3：ルール設定と Excel 出力

  1. 必要に応じて 資料タイトル・色ルール・変換キーワード・運転手 等を調整（各セクションは折りたたみ式）
  2. 「👁 プレビュー」で出力イメージを事前確認（推奨）
  3. 「📋 業務報告枠リスト」で業務報告イベントの時間枠一覧をテキストでコピー（任意）
  4. 「⬇ Excelをダウンロード」で保存
  5. 下部の処理ログで除外・変換・要確認などの結果を確認
    • 右上の「☐ 詳細表示」を ON にすると色判定根拠・除外理由などの詳細デバッグ情報が表示（不具合報告時に便利）

📋 業務報告枠リストとは？　出力日以降の平日について、業務報告イベントの時間枠を日付ごとに一覧化し、コピペ用のテキストとして出力します（例：5/19（月）9:00〜10:00）。45分以下の枠には （△） が付き、枠がない平日は「業務報告枠なし」と表示されます。「知事業務報告」は対象外（副知事分のみ）。各部署への調整依頼などに使えます。

4. ⚙️ 設定のカスタマイズ

設定はブラウザに自動保存されるため、次回起動時にも維持されます（同じブラウザに限る）。

📄 資料タイトル

Excel ファイルの各週ヘッダーに表示される名称です（例：「○○副知事週間日程表」）。空欄の場合はデフォルト（「週間日程表」）が使われます。

🎨 キーワード → セル色ルール

タイトルに特定キーワードを含む予定の背景色・文字色・強調（太字＋下線）を設定するルールです。背景色は白黒印刷でも文字がつぶれないパステル7色から選びます。

初期ルール（すべて編集可能）：

  • 「業務報告」→ パステル緑
  • 「移動」→ パステル黄・赤字・太字＋下線（ドライバー向けの視認性強化）
  • 「予定あり」→ パステル灰

自由設定：「✏️ ルールを編集する」から、キーワード・背景色・文字色・強調を追加・変更・削除できます。上から順に優先されます。どのルールにも一致しない予定はデフォルト色（初期値：パステル青）になります。

※「業務報告」「本会議・代表質問・一般質問」の説明欄から①②③…を抽出する機能は、色とは独立して動作します。

🔒 「予定あり」変換キーワード

プライバシー配慮のため、特定キーワードを含むタイトルを「予定あり」に置換します。配布用資料でセンシティブな予定名を隠す用途に使えます。

🌙 時間外OK扱いキーワード

17時以降のイベントは通常「📋 要確認」警告が処理ログに記録されます。ここに登録したキーワード（年度・大会・総会・新年会など）を含む予定は警告を抑止します。

🚗 担当運転手

A・B 2名の運転手名を設定し、月～金それぞれにどちらが担当かを割り当てできます。

自動の挙動：

  • 祝日・年末年始閉庁日（12/29〜1/3）に該当する曜日は運転手名を非表示
  • カレンダーに「○○運転手」「○○運転手対応」という終日予定を入れると、その日の運転手名を「○○」で上書き表示（一時的なローテーション変更に便利）
5. 🖨 印刷・PDF出力

出力した Excel を印刷

  1. ダウンロードした Excel ファイルを開く
  2. 「ファイル」→「印刷」を選択
  3. 推奨設定を確認：
    • 用紙：A4 横方向
    • 余白：「ユーザー設定の余白」（アプリで自動設定済み）
    • 拡大縮小：「拡大縮小なし」 ※自動最適化済みのため変更不要
  4. 「印刷」ボタンをクリック

PDF として保存

プリンター選択で「Microsoft Print to PDF」（Windows）または「PDFとして保存」（Mac）を選べば PDF 出力できます。

👁 プレビュー機能

Step 3 の「👁 プレビュー」ボタンで、Excel に書き出す前にブラウザ上で出力イメージを確認できます。新しいタブで A4 横サイズで表示され、そのまま印刷もできます。

⚠️ 印刷で右端や下端が切れる場合：印刷ダイアログの「拡大縮小」が「自動」になっていないか確認してください。「拡大縮小なし」に設定すると正しく印刷されます。

📖 マニュアルそのものを印刷

マニュアル右上の「🖨 マニュアルを印刷」ボタンから、全章を整形した A4 縦の印刷版を新タブで開けます。引き継ぎ資料・配布資料として利用できます。

6. 💾 バックアップと引き継ぎ

Step 3 下部の「⚙️ 設定の保存・復元」から、現在の設定を JSON ファイルとしてエクスポートできます。

エクスポートできる項目

  • 資料タイトル
  • 色ルール（デフォルト色含む）
  • 「予定あり」変換キーワード
  • 時間外OK扱いキーワード
  • 担当運転手（A・B 名と曜日割当）
  • プリセットカレンダー（メイン・2つ目）

※ ICS URL の保存設定・クライアントID・実績データは端末固有のため除外します。

引き継ぎ手順

  1. 元PC：「📤 設定をエクスポート」をクリック → JSONがダウンロードされる
  2. JSON ファイルを USB メモリやメール等で新担当者に渡す
  3. 新PC：「📥 設定をインポート」で JSON を読み込む
  4. 確認ダイアログで「OK」をクリック

⚠️ クライアントID は含まれません（PC ごとに個別設定）。新 PC では別途 Google Cloud Console から発行が必要です。

7. 🆘 トラブルシューティング

イベントが取得できない

  • 🔑 APIモード：クライアントID が正しく入力されているか、認証画面でログインしたかを確認。file://では動作しません
  • 📁 ICSファイルモード：ファイルが正しい .ics 形式か確認（zip 解凍後の中身を選択）
  • 🌐 ICS URLモード：CORS エラーで失敗する場合は ICSファイルモードに切替推奨。エラー詳細が画面に表示されます

不具合の調査・報告したい

  • 処理ログの右上にある「☐ 詳細表示」を ON にすると、各予定の色判定根拠・除外理由・場所補完の根拠などが表示されます
  • 「📋 コピー」ボタンで処理ログをクリップボードへ。開発者に共有する際に活用できます

一部の予定が Excel に出ない

  • 処理ログを確認（📅 除外された予定 🔁 重複・非表示 など）
  • 除外ルール（「※」始まり・「床屋」「散髪」を含む等）に該当していないか確認（※「ブロック」は除外されず「予定あり」に変換されます）
  • 同じ時間枠に3件以上重なっていないか確認

印刷で右側・下側が切れる

  • 印刷ダイアログで「拡大縮小」が「自動」になっていないか確認 →「拡大縮小なし」に
  • 「余白」が「狭い」「広い」等になっていないか → 「ユーザー設定の余白」のまま

設定が消えた

  • ブラウザ設定で localStorage がクリアされた可能性（履歴削除など）
  • 定期的に「設定をエクスポート」しておくと安心

認証エラーが出る（APIモード）

  • Google Cloud Console で、クライアントID の「承認済み JavaScript 生成元」にアプリを開く URL が登録されているか確認

業務報告・議会の項目（①②③）が表示されない

  • Googleカレンダーの説明欄に ①②③ または 1. 2. 3. 形式で項目が記載されているか確認
  • 予定の時間枠が30分未満の場合は表示されません（時間枠を広げると表示されます）
8. ❓ よくある質問（FAQ）

Q1. 3つのモード、どれを使えばいい？

A. サーバー経由（http(s)://）で開く運用なら、自動取得の 🔑 APIモード（または 🌐 ICS URLモード）が便利です。
HTMLをダブルクリックで開く（file://）運用なら、📁 ICSファイルモードが確実に動作します（毎回 .ics をダウンロードする手間はあります）。
初めて使う方には📁 ICSファイルモードがもっとも簡単です。

Q2. 色を変えるには？

A. Step 3 の「🎨 キーワード → セル色ルール」セクションを開き、「✏️ ルールを編集する」から変更できます。

Q3. 出力した Excel を編集してもいい？

A. もちろん編集可能です。通常の Excel ファイルなので、追記・修正・印刷設定の調整など自由に行えます。

Q4. 印刷時に内容が複数ページに分かれてしまう

A. 印刷ダイアログで「拡大縮小」を「拡大縮小なし」に設定してください。スケールは出力時に自動最適化済みです。

Q5. 業務報告の項目（①②③）が出ない

A. Googleカレンダーの説明欄に ①②③ または 1. 2. 3. 形式で記載してください。また、予定の時間枠が短い（30分未満）の場合は表示されません。

Q6. クライアントID はどこで確認できる？

A. Google Cloud Console の「APIとサービス」→「認証情報」で確認できます。一度入力すると次回以降も保存されます。

Q7. 過去のバージョンとの違いを知りたい

A. 画面右上のバージョン番号（例：v1.2.0）をクリックすると、これまでの更新履歴が表示されます。

Q8. このアプリ自体を別 PC にコピーするには？

A. gcal-excel-bundle.html ファイル1個をコピーするだけで動きます（インストール不要）。設定の引き継ぎは「設定の保存・復元」機能をご利用ください。

Q9. データはどこかに送信される？

A. すべてブラウザ内で処理されます。カレンダー情報は Google API から直接取得し、第三者サーバーには送信されません。

Q10. 動作するブラウザは？

A. 最新の Chrome / Edge / Firefox / Safari で動作確認しています。Internet Explorer は非対応です。

Q11. タイトルに「議員」を含む予定の警告が出る

A. 「議員」を含む予定は処理ログに「📋 議員含む・要確認」と通知されます。プライバシー配慮のため、必要に応じて「予定あり」変換キーワードに追加してください。

Q12. 「（三役）」付き予定の時間情報が出ない

A. タイトル末尾に「（三役）」を含めた上で、説明欄に「知事 10:00-10:30」「副知事 10:30-11:00」のような形式で時間を記載してください。セル末尾に「※知事 10:00-10:30 / 副知事 10:30-11:00」が追記されます。

Q13. 終日予定で運転手を上書きするには？

A. Googleカレンダーに「○○運転手」または「○○運転手対応」というタイトルで終日予定を入れてください。その日の運転手名が「○○」に上書きされます。

Q14. なぜか応接室と表示されない

A. v1.5.1 から、場所自動補完は「面会」「来客」（→応接室）と「幹部会」（→第二応接室）のみに限定されました。他の予定で場所欄が空のまま表示したくない場合は、カレンダー側で場所を入力してください。

9. 📝 Googleカレンダー入力のコツ（出力品質を高める書き方）

このアプリは Googleカレンダーのタイトル・場所・説明欄を読み取って Excel を生成します。入力のコツを押さえることで、自動色分けや自動補完が正しく機能し、修正の手間が大きく減ります。

📌 基本のルール

  • タイトル：1行で簡潔に。色分けやプライバシー変換のキーワード判定に使われます
  • 場所：住所または会議室名。空欄の場合、特定キーワード（面会・来客・幹部会）のみ自動補完されます
  • 説明欄：業務報告・議会系では ①②③ 形式で項目を列挙

✅ 良い入力例

業務報告（説明欄に部署を列挙）

タイトル：業務報告
場所：（空欄でOK）
説明：
①総務課
②政策企画課
③人事課
→出力：黄緑セル＋①②③で部署が列挙される

議会・一般質問（説明欄に質問項目を列挙）

タイトル：一般質問
説明：
①○○議員：人口減少対策について
②○○議員：観光振興について
→出力：青セル＋①②で議事項目が列挙される

面会

タイトル：○○氏 面会（場所空欄でも自動的に「応接室」になる）
または　○○氏 来客
→出力：「来客あり」に自動変換（プライバシー配慮）、場所に「応接室」自動補完

移動

タイトル：移動
場所：移動先（任意）
→出力：パステル黄セル＋赤字・太字＋下線（白黒印刷でも目立つ強調）

三役同席の会議

タイトル：○○会議（三役）
説明：
知事 10:00-10:30
副知事 10:30-11:00
→出力：セル末尾に「※知事 10:00-10:30 / 副知事 10:30-11:00」が追記される

運転手の臨時変更

タイトル：田中運転手 または 田中運転手対応（終日予定として登録）
→出力：その日の運転手名が「田中」に上書きされる
※通常は曜日ごとの固定割当ですが、終日予定でその日だけ変更可能

議員関連の予定（注意喚起あり）

タイトル例：○○議員 面会
→処理ログに「📋 議員含む・要確認」と表示される
プライバシー配慮で「予定あり」変換したい場合は、設定の「🔒 「予定あり」変換キーワード」に「議員」を追加するか、個別キーワード（議員名等）を登録

⚠️ 間違いやすい入力例

❌ 半角カナが混入

悪い例：ｵｵﾆｼ会長 面会
良い例：大西会長 面会
→処理ログに「⚠️ 誤植の可能性」と記録されます

❌ 数字の全半角混在

悪い例：第1回○○会議（第１期）
良い例：第1回○○会議（第1期） または 全角統一
→処理ログに警告

❌ 業務報告の項目を「・」「-」で列挙

悪い例：
・総務課
・政策企画課
良い例：
①総務課
②政策企画課
→丸数字でないと項目として抽出されません

❌ タイトルが「※」で始まる（除外対象）

※確認中：○○打合せ のような「※」始まりの予定はExcelに出力されません。除外したくない予定は別の表現にしてください。
なお ブロック を含む予定は除外されず、「予定あり」に変換して出力されます（注記なし）。

💡 ちょっとした工夫

  • 非公開にしたい予定（プライベート用通院など）→ 「予定あり」変換キーワードに登録すれば自動的に「予定あり」と表示される
  • 17時以降の懇親会など → 「時間外OK扱いキーワード」に登録すれば「📋 要確認」警告が抑止される
  • 運転手を一時的に変更したい日 → その日に終日予定で ○○運転手 または ○○運転手対応 を入れる
  • 祝日・年末年始（12/29〜1/3）の運転手表示 → 自動的に非表示。手動の終日予定で上書き可能
  • 場所欄を空にしておきたい予定 → タイトルに「面会」「来客」「幹部会」を入れない（v1.5.1 以降、これら以外は補完なし）
10. 📁 ICSファイルのダウンロード手順（詳細）

APIモードが使えない場合（file:// で開いた場合など）の代替手段として、Googleカレンダーから .ics ファイルをダウンロードして利用できます。

方法A：カレンダー単体をエクスポート（推奨）

  1. ブラウザで Googleカレンダー を開く
  2. 左サイドバーで、エクスポートしたいカレンダーにマウスを乗せる
  3. カレンダー名の右に表示される「⋮」（縦三点メニュー）をクリック
  4. 「設定と共有」を選択
  5. 左メニューで「カレンダーの設定」セクション内の「カレンダーのエクスポート」をクリック
  6. zip ファイルがダウンロードされる
  7. zip を解凍すると、カレンダー名@gmail.com.ics のような .ics ファイルが取り出せる

方法B：すべてのカレンダーを一括エクスポート

  1. Googleカレンダー右上の「歯車アイコン ⚙️」→「設定」
  2. 左メニュー「インポート / エクスポート」
  3. 「エクスポート」セクションで「エクスポート」ボタンをクリック
  4. すべてのカレンダーが含まれた zip がダウンロードされる

本アプリでの読み込み

  1. アプリ Step 1 で「📁 .icsファイル」タブを選択
  2. 「📂 ファイルを選択」で .ics ファイルを選ぶ
  3. 必要に応じて「📂 追加ファイルを選択」で2つ目の .ics を追加
  4. 「✅ 読み込む」をクリック

💡 運用Tip：毎週ダウンロード→読み込みする場合、エクスポート手順をブックマークしておくと素早く操作できます。
